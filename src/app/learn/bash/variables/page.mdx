# Bash Variables

## Introduction

As in any other programming language, you can use variables in Bash Scripting as
well. However, there are no data types, and a variable in Bash can contain
numbers as well as characters.

## Variables

### Basic Variable Assignment

To assign a value to a variable, all you need to do is use the `={:bash}` sign:

```bash
name="XLearn"
```

> [!NOTE]
>
> You can not have spaces before and after the `={:bash}` sign.

After that, to access the variable, you have to use the `${:bash}` and reference
it as shown below:

```bash
echo $name
```

Wrapping the variable name between curly brackets is not required, but is
considered a good practice, and I would advise you to use them whenever you can:

```bash
echo ${name}
```

The above code would output: `XLearn{:txt}` as this is the value of our name
variable.

### Using Variables in Scripts

Next, let's update our `hello.sh{:bash}` script and include a variable in it.
Open the file with your favorite text editor. Now add the name variable here in
the file, with a welcome message. Our file now looks like this:

```bash {3,4}
#!/bin/bash

name="Tom"
echo "Hi there $name"
```

Save the file and run it. Use the instructions from the
[Bash Hello World](/learn/bash/hello) tutorial incase you forgot how to run or
save the file. You should see the following output on your screen:
`Hi there Tom{:txt}`.

Here is a rundown of the script written in the file:

- **`#!/bin/bash{:bash}`:** At first, we specified our Shebang.
- **`name="Tom"{:bash}`:** Then, we defined a variable called `name{:bash}` and
  assigned a value to it.
- **`echo="Hi there $name"{:bash}`:** Finally, we output the content of the
  variable on the screen as a welcome message by using `echo{:bash}`.

### Command Line Parameters

You can also add multiple variables in the file as shown below. If you save this
file and run it you'll see `Hello Tom{:txt}` on the terminal.

```bash {3,4}
#!/bin/bash

name="Tom"
greeting="Hello"

echo "$greeting $name"
```

> [!NOTE]
>
> Note that you don't necessarily need to add semicolon `;{:bash}` at the end of
> each line. It works both ways, a bit like other programming language such as
> JavaScript!

You can also add variables in the Command Line outside the Bash script and they
can be read as parameters:

```bash
./devdojo.sh Gilfoyle buddy!
```

This script takes in two parameters `Gilfoyle{:txt}` and `buddy!{:txt}`
separated by space. In the `hello.sh{:bash}` file we have the following:

```bash {/$1/, 3}
#!/bin/bash

echo "Hello there" $1
```

`$1{:bash}` is the first input `Gilfoyle{:txt}` in the Command Line. Similarly,
there could be more inputs and they are all referenced to by the `${:bash}` sign
and their respective order of input. This means that `buddy!{:txt}` is
referenced to using `$2{:bash}`. Another useful method for reading variables is
the `$@{:bash}` which reads all inputs. So now let's change the
`hello.sh{:bash}` file to better understanding.

```bash {/$1/, /$2/, /$@/, 4, 7, 10}
#!/bin/bash

# $1 : first parameter
echo "Hello there" $1

# $2 : second parameter
echo "Hello there" $2

# $@ : all
echo "Hello there" $@
```

This following script will output the follwing to the terminal.

```bash
Hello there Gilfoyle
Hello there buddy!
Hello there Gilfoyle buddy!
```

## Conclusion

Variables are fundamental building blocks in Bash scripting. They allow you to
store and manipulate data, making your scripts more dynamic and reusable.
Whether you're using basic variable assignments or working with command line
parameters, understanding how variables work is essential for effective Bash
scripting.
